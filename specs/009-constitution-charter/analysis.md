# Analysis: Constitution as Charter

## Summary

spec 009 is internally consistent and well-covered: all 8 FRs and 6 SCs map to concrete tasks with files, verification, and dependencies; no orphan components or tasks; the file-safety order is correct; the Constitution Check is honest (the spec 007 "regenerable tooling" classification is refined via governance, not silently violated). One **critical** finding — the status detector's classification logic as written does not cover a spec-listed edge case (the zero-principle "authored shell") — plus five advisories. The critical item is a small plan/tasks logic refinement (not a structural change) and should be resolved before T002 is implemented. With that fix, the spec is ready for `/spec-implement`.

## Issues Found

### Critical

- **C1 — Detector classification leaves the "authored shell" edge case unclassified.** The spec's Edge Cases explicitly require: *"Constitution present but zero principles (authored shell): treat as AUTHORED-but-thin → amend branch."* But the plan (C1) and task T002 define the detector as: signature tokens present → `TEMPLATE`; **else ≥1 `### ` principle → `AUTHORED`**; file missing → `ABSENT`. A file with **no signature tokens AND zero principles** (the authored shell) satisfies none of the three branches — it is not `TEMPLATE` (no sig tokens), not `AUTHORED` (needs ≥1 principle per the rule as written), and not `ABSENT` (it exists). If T002 is implemented literally, the authored-shell edge case gets no status and the skill takes no defined branch — implementation misses a spec edge case.
  - **Fix (logic refinement, not structural)**: redefine the discriminator so principle *count* is informational, not gating — `TEMPLATE` = ≥1 signature token present; `AUTHORED` = no signature tokens present (report the principle count alongside, which may be 0); `ABSENT` = file missing. The authored shell (no sig tokens, 0 principles) then classifies as `AUTHORED` → amend branch, exactly as the spec intends.
  - **Touch**: plan C1 detector logic, task T002 detector rules, and the spec's Key Entities "Constitution status" definition (which currently says AUTHORED requires "≥1 principle"). Resolve before T002.

### Advisory

- **A1 — SC-001 / US1 overstates a coaching-enforced guarantee.** SC-001 says *"settled principle text cannot drift on a re-run,"* but FR-001's amend-guard is **skill-coaching only** — there is no mechanical backstop (unlike US3/FR-006, which is enforced in `uninstall.ts` code). The T003 content-assertion proves the amend *guidance is present*, not that the AI *follows* it. This is philosophy-consistent (Constitution II — Coach-Not-Gatekeeper; the tool coaches, the AI exercises judgment), but the success criterion claims hard prevention for a best-effort mechanism. **Suggest**: soften SC-001 to "the amend guidance is present and the skill branches on AUTHORED status (coaching-enforced)" and note the asymmetry with US3 (mechanically enforced), so the spec is honest about which guarantees are code-backed vs coaching-backed. (Optional stronger option: an advisory pre-write diff check that warns — never blocks — when a non-targeted principle's text changes; decide whether that crosses from coach toward gatekeeper.)

- **A2 — Installed-skill regeneration command is unspecified.** T003/T004 say "regenerate the installed `.claude/skills/spec-constitution/SKILL.md` from source" but do not name the command. Pin it so the tracked installed copy is actually synced and not forgotten — e.g., `npx tsx src/cli.ts agents update` (re-installs all skills for the default agent, idempotent), or a one-off `installSkill` invocation. The plan (C2) says "regenerate" abstractly.

- **A3 — T005's `corpus-uninstall.test.ts` regression watchpoint is slightly mis-stated.** T005 says "update that suite's expectations if it asserted the old behavior." Concrete check: `tests/units/corpus-uninstall.test.ts:61` asserts `"constitution REMOVED on plain uninstall (regenerable tooling)"` (`!exists(... constitution.md)`). Its fixture is a **template** constitution (created via `init`/`installConstitutionToMemory`; no test hand-writes an authored constitution — confirmed by grep). Under T005's new logic a **template** is still removed → **the assertion still passes; there is no functional break.** Two real actions though: (a) the test's name/premise *"regenerable tooling"* is now stale/misleading — reframe it (e.g., "template constitution removed on plain uninstall"); (b) it does not cover the new AUTHORED-preserve behavior (SC-003), which T005's new tests must add. Downgrade T005's "possible regression" to "reframe existing assertion + add authored-preserve coverage."

- **A4 — Re-authored charter version not pinned.** FR-008 / T006 say "set the footer version per FR-004 semantics, continuing the v1.4.0 lineage" without pinning the number. It is derivable — the 5 principles are unchanged (not MAJOR) and the charter-as-IP clause is added/expanded (MINOR) → **v1.5.0** — but pin it in T006 so the implementer doesn't guess.

- **A5 — `--reset` escape hatch is an untested promise.** FR-001 / US1 acceptance #3 references an explicit `--reset` opt-in as the only path to a full rewrite of an AUTHORED constitution; T003 mentions it. T003's content-assertion should explicitly verify the skill documents the `--reset` escape hatch, else it is a stated behavior with no verification.

- **FR-007 "already true → codify + test only" is a correct, code-avoiding insight.** Confirming `installConstitutionToMemory` is called only from `init.ts:105` (with the `if exists return false` guard) and `update.ts` "Never modifies user artifacts" turns C4/T007 into a comment + regression tests instead of new logic — appropriately minimal.
- **Testing discipline matches deliverable type.** Behavioral tasks (T002/T005/T007) are RED-first; skill-prose (T003/T004) and the re-author (T006) use content-assertion — the right tool for markdown coaching (Constitution V), without pretending to simulate AI behavior.
- **Status-aware uninstall precisely honors US3.** C3/T005 preserve an AUTHORED constitution and remove only a TEMPLATE, rather than a crude "always preserve" — exactly the spec's tier-crossing semantics (tooling→IP at authoring time).
- **`.claude/` tracking wrinkle is surfaced and handled** (regenerate the installed copy from source, not hand-edit, preserving the source→installed derivation).
