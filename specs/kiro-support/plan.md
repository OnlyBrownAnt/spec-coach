## Summary

Add Kiro as a supported AI agent in spec-coach. Kiro uses the `skills` format (same family as Claude Code and Codex): each skill is a directory under `.kiro/skills/<skill-id>/SKILL.md` with YAML frontmatter. The implementation is a single `AgentConfig` entry in the `AGENTS` map plus one type union update. No new logic, no new files, no changes to skill content or templates.

### Kiro AgentConfig Entry

```typescript
kiro: {
  key: "kiro",
  name: "Kiro",
  dir: ".kiro/skills",
  format: "skills",
  separator: "-",
  frontmatter: {},
},
```

**Design decisions**:
- `frontmatter: {}` (empty): Kiro auto-activates skills by matching `description` against user requests. The `description` is always generated by `buildSkillFrontmatter()` from the source skill's existing `description` frontmatter. No extra fields needed.
- No `argumentHints`: Kiro natively supports `$ARGUMENTS` passthrough. Argument hints are a Claude Code metadata convention, not applicable here.
- No `allowed-tools` or `model` frontmatter: These are optional Kiro fields that the user can add to their own SKILL.md if desired. Spec-coach doesn't set them.
- `separator: "-"`: Produces skill IDs like `spec-specify`, matching Kiro's slash command format (`/spec-specify`).
- `format: "skills"`: Triggers the directory-per-skill installation path used by Claude Code and Codex. This path is already tested.
